- 2026-07-17: Fixed SQLITE_BUSY (database is locked) on POST /boards. Root cause: Hibernate Envers `@GeneratedValue` on NelloRevision.id used the default TABLE strategy, opening an isolated second DB connection to generate revision IDs — SQLite's single-writer model causes BUSY when the main transaction holds the lock. Fix: (1) Changed to `GenerationType.IDENTITY` so SQLite auto-increment handles IDs with no extra connection. (2) Added SqliteDataSourceConfig bean setting `PRAGMA journal_mode=WAL` (persistent, enables concurrent reads) and HikariCP `connectionInitSql=PRAGMA busy_timeout=30000` (per-connection retry for 30s). (GitHub Copilot / claude-sonnet-4.6)
